# Backend SPECS

Local SDD artifacts:

- `PRD.md`
- `ARCHITECTURE.md`
- `API.md`
- `FEATURES/<feature-slug>/spec.md`
- `FEATURES/<feature-slug>/tasks.md`

`API.md` references the root API contract. Do not duplicate the cross-end API contract here.

Use `kit sdd <feature-slug>` from the project root to create a feature skeleton. Keep feature specs aligned with `workflow/solution-selected.md`, root `SPECS/API.md`, and root `SPECS/DATABASE.md`.
