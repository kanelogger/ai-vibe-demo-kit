# 用户管理模块需求

## 1. 模块目标

用于维护系统可登录用户的基础资料、组织归属、岗位、状态和角色集合。

## 2. 查询条件

- 用户工号
- 登录名
- 中文姓名
- 手机号
- 所属部门
- 岗位
- 用户状态

## 3. 列表字段

- 用户工号
- 登录名
- 中文姓名
- 手机号
- 邮箱
- 所属部门
- 岗位
- 用户状态
- 创建时间
- 操作

## 4. 维护字段

- 用户工号
- 登录名
- 密码
- 中文姓名
- 手机号
- 邮箱
- 所属部门
- 岗位
- 用户状态
- 角色集合

## 5. 操作规则

- 支持新增、编辑、查看、启用、停用、重置密码和删除用户
- 用户工号必须唯一
- 登录名必须唯一
- 密码必须加密存储
- 一个用户可以拥有多个角色
- 停用用户不允许登录
- 用户删除采用逻辑删除，历史日志仍可展示用户名称

## 6. 角色分配规则

- 用户维护界面支持给用户分配多个角色
- 已停用角色不可继续分配给新用户
- 修改用户角色后，该用户下次获取当前用户信息时应返回新的角色和菜单范围

## 7. Demo 要求

- 用户列表支持分页、查询和重置
- 新增、编辑、启用、停用、重置密码操作可交互
- 修改用户角色后切换该账号登录，能体现菜单权限变化
