# 部门管理模块需求

## 1. 模块目标

用于维护用户所属部门，为人员归属、查询筛选和后续数据范围控制提供基础数据。

## 2. 查询条件

- 部门编码
- 部门名称
- 部门状态

## 3. 列表字段

- 部门编码
- 部门名称
- 部门状态
- 创建时间
- 操作

## 4. 维护字段

- 部门编码
- 部门名称
- 部门状态
- 部门说明

## 5. 业务规则

- 第一阶段只支持单层级部门结构
- 部门编码必须唯一
- 停用部门不可分配给新增用户
- 已被用户引用的部门不允许物理删除
- 历史用户和日志需要继续展示部门名称
- 后续可扩展为多级部门结构和数据范围控制

## 6. Demo 要求

- 支持新增、编辑、启用、停用和删除部门
- 用户管理中的部门下拉选项来自部门数据
- 停用部门后不应出现在新增用户表单中
