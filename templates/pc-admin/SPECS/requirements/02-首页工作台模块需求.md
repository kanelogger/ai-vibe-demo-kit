# 首页工作台模块需求

## 1. 模块目标

首页用于展示当前登录用户的工作概览，包括待办数量、未读消息、最近操作、系统公告和管理员统计数据。

## 2. 页面结构

- 顶部栏：展示系统名称、当前登录用户、消息入口和退出操作
- 左侧菜单栏：按当前用户可访问菜单动态展示
- 中间内容区：展示工作台统计、待办、消息、公告和最近操作
- 管理员统计区：仅管理员可见

## 3. 展示内容

- 我的待办数量
- 未读消息数量
- 最近操作记录
- 系统公告列表
- 管理员统计：用户数、角色数、菜单数、今日登录数、接口异常数

## 4. 业务规则

- 首页统计按当前登录用户权限范围展示
- 普通用户不能看到管理员统计区
- 未读消息数量来自消息中心
- 最近操作记录来自操作日志
- 系统公告作为首页展示区块，不单独强制建设公告维护模块

## 5. Demo 要求

- 首页加载后展示真实接口返回数据，不使用前端 Mock
- 切换账号后首页统计和菜单展示应发生变化
- 点击未读消息可进入消息中心
- 最近操作应能反映当前账号的新增、修改、删除等关键动作
