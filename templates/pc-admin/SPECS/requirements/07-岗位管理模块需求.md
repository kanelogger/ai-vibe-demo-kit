# 岗位管理模块需求

## 1. 模块目标

用于维护人员岗位信息，描述用户在组织中的职责分类，并为用户资料、筛选和后续业务规则提供基础数据。

## 2. 查询条件

- 岗位名称
- 岗位编码
- 岗位状态

## 3. 列表字段

- 岗位名称
- 岗位编码
- 岗位状态
- 岗位说明
- 创建时间
- 操作

## 4. 维护字段

- 岗位名称
- 岗位编码
- 岗位状态
- 岗位说明

## 5. 业务规则

- 岗位编码必须唯一
- 常见岗位包括管理员、运营人员、审核人员、普通用户
- 停用岗位不可分配给新增用户
- 已被用户引用的岗位不允许物理删除
- 岗位不直接决定菜单权限，菜单权限仍由角色控制

## 6. Demo 要求

- 支持新增、编辑、启用、停用和删除岗位
- 用户管理中的岗位下拉选项来自岗位数据
- 停用岗位后不应出现在新增用户表单中
