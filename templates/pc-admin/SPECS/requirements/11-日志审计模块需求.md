# 日志审计模块需求

## 1. 模块目标

记录用户登录、关键业务操作、接口访问异常和系统错误，为问题排查、合规审计和操作追踪提供依据。

## 2. 页面结构

- 登录日志页
- 操作日志页
- 异常日志页

## 3. 登录日志字段

- 登录用户
- 登录名
- 登录时间
- 登录 IP
- 登录结果
- 失败原因

## 4. 操作日志字段

- 操作人
- 操作时间
- 操作模块
- 操作类型
- 请求方法
- 请求路径
- 请求参数
- 操作结果
- 错误信息

## 5. 异常日志字段

- 发生时间
- 接口路径
- 请求方法
- 错误类型
- 错误信息
- 堆栈摘要
- 处理状态

## 6. 业务规则

- 登录成功和登录失败均需记录登录日志
- 新增、修改、删除、审核、导入、导出等关键动作需记录操作日志
- 接口异常和系统错误需记录异常日志
- 敏感字段如密码、Token 不得明文写入日志
- 日志默认只允许查询，不允许普通用户删除或修改
- 超级管理员可按时间范围查询日志

## 7. Demo 要求

- 登录、退出、用户维护、角色维护、菜单授权等动作能生成操作日志
- 日志列表支持查询、分页和详情查看
- 异常日志可通过模拟失败接口或错误操作展示
