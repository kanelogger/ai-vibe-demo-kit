# 登录认证模块需求

## 1. 模块目标

为后台管理系统提供统一登录、退出、Token 刷新和当前用户初始化能力，保障所有非公开功能必须在登录态下访问。

## 2. 页面结构

- 登录页：包含系统名称、登录名、密码和登录按钮
- 登录失败提示区：展示账号禁用、密码错误等明确提示
- 登录成功后进入首页工作台
- 顶部栏提供退出登录入口

## 3. 表单字段

- 登录名
- 密码
- 记住登录状态

## 4. 业务规则

- 用户输入登录名和密码后完成身份认证
- 用户状态为停用时不允许登录
- 密码必须按加密后的值校验，数据库不得保存明文密码
- 登录成功后返回 Access Token 和 Refresh Token
- 后续接口调用必须在请求头中携带 Access Token
- Access Token 过期后允许使用 Refresh Token 换取新的 Access Token
- 退出登录后前端必须清除本地 Token、用户信息、角色和菜单缓存

## 5. 初始化规则

- 前端启动后调用当前用户信息接口
- 当前用户信息接口返回用户基础信息、角色集合和可访问菜单树
- 前端根据菜单树动态注册路由并渲染侧边栏
- Token 无效、刷新失败或用户被停用时跳转回登录页

## 6. Demo 要求

- 提供至少一个超级管理员账号和一个普通用户账号
- 能验证登录成功、密码错误、账号停用和退出登录
- 能通过切换账号体现菜单权限差异
- 所有业务页面必须依赖登录态访问
